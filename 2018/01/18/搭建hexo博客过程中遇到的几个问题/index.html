搭建hexo博客过程中遇到的几个问题

发表于 2018-01-18
字数统计:

浏览了一些hexo博客，发现很多作者的第一篇文章都是讲建站过程，说明hexo的用户大多和我一样是初学者呢～下面我总结了下在跟着教程操作时遇到的几个问题以及解决过程：

安装不了brew

#安装
cd ～
ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

按照上面的代码输入安装命令之后，显示

#报错
Password:
...
==> Searching online for the Command Line Tools
==> /usr/bin/sudo /usr/bin/touch /tmp/.com.apple.dt.CommandLineTools.installondemand.in-progress
不能连接到软件更新服务器（upapple.sankuai.info）。
==> Installing
==> /usr/bin/sudo /usr/sbin/softwareupdate -i
Software Update Tool

: No such update
No updates are available.
...
Failed during: /usr/bin/sudo /usr/bin/xcode-select --switch /Library/Developer/CommandLineTools

查了下原因发现是因为没有安装命令行工具，可是苹果自定义安装时提示无法连接网络（这是因为苹果把这个工具给移除了）。
最后手动在Apple developer 下载了command line tools

hexo d上传之后GitHub的文件和本地文件不一致问题

上传时本地的文件目录为
.
├── README.md
├── _config.yml
├── db.json
├── debug.log
├── gulpfile.js
├── index.html
├── node_modules
├── package-lock.json
├── package.json
├── public
├── scaffolds
├── sitemap.xml
├── source
└── themes

但上传到GitHub里就只有public文件夹的内容了:
.
├── 2018
├── 404.html
├── archives
├── css
├── images
├── index.html
├── js
├── lib
├── offline.html
└── sitemap.xml

幸好打开页面还是能用的～done
